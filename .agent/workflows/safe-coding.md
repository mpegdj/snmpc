# 안전한 코딩 규칙

이 규칙은 코드 파손을 방지하고 복구 가능성을 확보하기 위한 필수 규칙입니다.

## 🚨 필수 규칙

### 1. 커밋 체크 규칙
- **수정된 파일(Modified)이 5개 이상 쌓이면 반드시 멈추고 커밋할지 물어볼 것**
- 더 이상 진행하지 말고 사용자에게 먼저 확인받을 것
- `git status`로 현재 상태 확인 후 보고

### 2. 사전 허락 규칙
- 코드 수정 전 반드시 사용자 허락 받기
- 여러 파일 동시 수정 금지 - 한 파일씩
- 빌드 확인 후 다음 파일로 넘어가기

### 3. 안전장치
- 큰 작업 시작 전 현재 상태 커밋 권유
- 빌드 실패 시 즉시 멈추고 보고
- 실행 테스트 실패 시 즉시 멈추고 보고

## 📋 작업 흐름

1. 작업 계획 → 사용자 승인
2. 파일 1개 수정 → 빌드 확인
3. 성공 시 → 다음 파일 (단, Modified 5개 미만일 때만)
4. Modified 5개 이상 → **멈추고 커밋 권유**
5. 커밋 완료 → 다음 단계 진행

## ⚠️ 금지 사항

- 허락 없이 마음대로 진행
- 여러 파일 동시 수정
- 커밋 없이 장시간 작업
- 빌드 실패 상태에서 추가 작업
