# SnmpNms GUI를 VS Code 스타일로 변경하는 계획

## 개요

SnmpNms 애플리케이션의 GUI를 VS Code 스타일로 변경하여 현대적이고 일관된 사용자 경험을 제공합니다.

**현재 상태**: Phase 1, 2 완료 ✅
- Activity Bar, Sidebar, BottomPanel 구현 완료
- 레이아웃 통합 완료
- 밝은 테마 적용 완료
- 모든 기능 정상 동작 확인

---

## 현재 구조 분석

### 현재 레이아웃
- **상단**: Menu Bar + ToolBar (2개)
- **좌측**: Selection Tool (GroupBox, TabControl)
- **중앙**: View Window Area (TabControl)
- **하단**: StatusBar + Event Log

### 현재 컴포넌트
- `MainWindow.xaml`: 메인 창 레이아웃
- `Selection Tool`: Map Tree, Device List, Trend, Event, Menu 탭
- `View Window Area`: Map View, Device Details, MIB Tree, MIB Table, MIB Graph, SNMP Test, Event Log 탭
- `Event Log`: 하단 고정 영역

```xml
      <RowDefinition Height="*"/>
      <RowDefinition Height="5"/>

    </Grid.RowDefinitions>
  </Grid>
</DockPanel>
```

```xml
<DockPanel>
```

### 해결한 문제

1. **Panel 클래스 이름 충돌**
   - 문제: WPF의 `Panel` 클래스와 충돌
   - 해결: `BottomPanel`로 클래스명 변경

2. **데이터 바인딩 문제**
   - 문제: Sidebar의 ContentControl 바인딩으로 인해 DataContext 전달 안 됨
   - 해결: DependencyProperty의 PropertyChangedCallback 사용하여 직접 Content 설정

3. **트리뷰 텍스트 미표시**
   - 문제: 어두운 배경에 어두운 텍스트로 인해 보이지 않음
   - 해결: 밝은 테마로 변경, 텍스트 색상 검은색으로 설정

4. **Activity Bar 아이콘 미표시**
   - 문제: 어두운 배경에 어두운 아이콘으로 인해 보이지 않음
   - 해결: 배경색 밝게 변경, 아이콘 색상 검은색으로 설정, 활성화 시 흰색

5. **MIB 트리 참조 오류**
   - 문제: 기존 `treeMib` 참조가 제거되어 빌드 오류 발생

## 기술적 세부사항

### DependencyProperty 사용
- `Sidebar.HeaderText`: 헤더 텍스트
- `Sidebar.CurrentContent`: 동적 콘텐츠
- PropertyChangedCallback을 통한 ContentControl 업데이트

### 이벤트 전달 메커니즘
- `SidebarMapView`의 TreeView 이벤트를 MainWindow로 전달
- `MapNodeTextMouseLeftButtonDown` 이벤트 구현

### 데이터 바인딩
- `SidebarMapView.DataContext` = `MainViewModel`
- `ItemsSource="{Binding MapRoots}"` 바인딩
- `Loaded` 이벤트에서 초기화하여 타이밍 문제 해결

### 스타일 상속
- `ActivityBarButtonActiveStyle`은 `ActivityBarButtonStyle` 기반
- `ActivityBarIconActiveStyle`은 `ActivityBarIconStyle` 기반
- `VSCodeTreeViewItemStyle`을 TreeViewItem에 적용
