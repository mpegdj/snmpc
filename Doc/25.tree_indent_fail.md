# MIB Tree Indent 설정 작업 기록

## ✅ 최종 적용된 변경 사항

### `VSCodeTheme.xaml` 수정

**위치**: `SnmpNms.UI/Resources/VSCodeTheme.xaml`

1. **`ExpanderColumn`에 `x:Name` 추가** (191번째 줄)
   ```xml
   <ColumnDefinition x:Name="ExpanderColumn" Width="Auto"/>
   ```

2. **`HasItems`가 False일 때 `ExpanderColumn` Width를 0으로 설정** (244번째 줄)
   ```xml
   <Trigger Property="HasItems" Value="False">
       <Setter TargetName="Expander" Property="Visibility" Value="Hidden"/>
       <Setter TargetName="ExpanderColumn" Property="Width" Value="0"/>
   </Trigger>
   ```

### 결과
- ✅ **자식 없는 노드의 Expander 공간이 제거되어 시각적으로 더 컴팩트해짐**
- ❌ **Indent는 줄지 않음** (여전히 8px로 보임)

### 적용 범위
- Map tab view (SidebarMapView)
- MIB tab view (SidebarMibView)
- 모든 `VSCodeTreeViewItemStyle`을 사용하는 TreeView

## 참고: Indent 줄이기 시도 실패

원래 목표였던 Indent를 8px에서 6px로 줄이기는 실패했습니다. 여러 방법을 시도했지만:
- `ItemsPresenter`의 `Margin` 변경
- 리소스 오버라이드
- ControlTemplate 완전 오버라이드
- 등등...

모두 실패했으며, 실제 indent는 `ItemsPresenter`의 `Margin`이 아니라 다른 곳에서 결정되는 것으로 보입니다.

코드에서 불필요한 부분을 제거하고 성공한 내용만 남기겠습니다. 관련 파일 확인 중입니다.

## 정리된 내용

### 유지한 것 (성공한 변경)
1. **`ExpanderColumn`에 `x:Name` 추가** (191번째 줄)
2. **`HasItems`가 False일 때 `ExpanderColumn` Width를 0으로 설정** (244번째 줄)

### 제거/복원한 것
- **`ItemsPresenter`의 `Margin`을 원래대로 복원**
  - `Margin="4,0,0,0"` → `Margin="{StaticResource VSCodeTreeIndent}"` (원래대로)
  - Indent가 줄지 않았으므로 원래 리소스 참조로 복원

## 최종 상태

- `ExpanderColumn` x:Name 추가됨
- `HasItems` False일 때 `ExpanderColumn` Width = 0 설정됨
- `ItemsPresenter` Margin은 원래 리소스 참조로 복원됨

코드가 정리되었고, 성공한 변경만 남았습니다.
